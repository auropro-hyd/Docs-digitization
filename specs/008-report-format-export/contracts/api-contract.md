# API Contract: Client-Aligned Compliance Report (Spec 008)

## Surface area

Two endpoints change, one stays the same with new content, two are new.

| Method | Path | Status | Change |
|---|---|---|---|
| `GET` | `/api/compliance/{doc_id}/report` | unchanged shape | underlying data unchanged; just sourced for the new view |
| `GET` | `/api/compliance/{doc_id}/export` | **breaking content; same URL** | rewrites HTML/MD renderers; adds `format=pdf` |
| `GET` | `/api/compliance/{doc_id}/report-rows` | **new** | returns the rule-centric `ReportDocument` JSON for the frontend |
| `POST` | `/api/compliance/{doc_id}/mitigation/synthesize` | **new** | warms the mitigation-text cache for findings missing recommendation |
| `GET` | `/api/compliance/{doc_id}/preview` | **new** | server-side PDF render returned inline (for in-browser preview before download) |

## `GET /api/compliance/{doc_id}/export`

**Existing endpoint, content overhauled.**

### Query parameters

| Name | Type | Required | Default | Notes |
|---|---|---|---|---|
| `format` | `"pdf" \| "html" \| "md"` | no | `"pdf"` | NEW default; was implicit HTML before |
| `agent` | string | no | (all) | when supplied, only rows for that agent appear in the export |

### Response

- `200 OK` ŌĆö body is the rendered file; `Content-Disposition: attachment; filename="<doc>_compliance_report.<ext>"`; `Content-Type` per format (`application/pdf`, `text/html`, `text/markdown`).
- `404 Not Found` ŌĆö no `compliance_result.json` for the doc_id (compliance hasn't been run).
- `503 Service Unavailable` ŌĆö renderer crashed AND fallback also failed. Body carries `{"error": "...", "trace_id": "..."}`.

### Behaviour requirements

- Reads `compliance_result.json` for the doc_id; renders via `report_renderer.render(report_document, format=...)`.
- When `format=pdf` and the renderer raises, falls back to HTML with a `X-Render-Fallback: html` header. Never silent-fails.
- Always emits `compliance.report_rendered` telemetry: `doc_id`, `format`, `row_count`, `compliant_count`, `action_required_count`, `needs_attention_count`, `render_duration_ms`, `agent_filter`.
- Idempotency: two consecutive calls with the same `(doc_id, format, agent)` produce byte-identical output except for the `Printed On` timestamp in the footer.
- Caching: writes the rendered file to `data/documents/{doc_id}/exports/report.{ext}` after successful render. Subsequent calls on the same `compliance_result.json` mtime return the cached file unless `?nocache=1`.

## `POST /api/compliance/{doc_id}/mitigation/synthesize` *(new)*

Operator-triggered batch synthesis of mitigation text for findings whose `recommendation` is empty. Eagerly populates the cache so the next export doesn't take an unbounded number of LLM calls during the render path itself.

### Request body

```json
{
  "rule_ids": ["CHE-BPC11", "GMP-EQU11"],   // optional; defaults to "all findings missing mitigation"
  "force": false                            // optional; when true, re-synthesises even cached
}
```

### Response

```json
{
  "synthesised_count": 2,
  "skipped_count": 19,
  "rule_ids_touched": ["CHE-BPC11", "GMP-EQU11"],
  "total_cost_estimate_usd": 0.04
}
```

### Behaviour

- Loads `compliance_result.json`; iterates the findings list; for each finding without `recommendation` AND (without `mitigation_text` OR `force=true`), calls the synthesis LLM once.
- Persists the result back to `compliance_result.json` (atomic write to `.tmp` then rename).
- Emits `compliance.mitigation_synthesised` telemetry per finding: `rule_id`, `agent`, `cost_estimate_usd`, `duration_ms`.
- Enforces per-export cost cap (FR-SC-006): rejects with `429 Too Many Requests` if the estimated cumulative spend for the request exceeds `$0.50`.

## `GET /api/compliance/{doc_id}/preview` *(new)*

Returns the rendered PDF inline (`Content-Disposition: inline`) so the frontend can iframe it for a preview before the operator commits to downloading. Same renderer as `/export?format=pdf`.

### Query parameters

| Name | Type | Required | Default |
|---|---|---|---|
| `agent` | string | no | (all) |

### Response

- `200 OK` ŌĆö PDF bytes with `Content-Disposition: inline; filename="<doc>_preview.pdf"`.
- `404 Not Found` ŌĆö no compliance result yet.

## `GET /api/compliance/{doc_id}/report` (unchanged)

The raw `ComplianceReport` JSON. Kept exactly as it is today so existing consumers (downstream services, dashboards) keep working. The frontend stops consuming this for the rule-centric view (switches to `/report-rows`) but the endpoint still serves any client that wants the full structured shape.

## Quiet routes

The new `/report-rows` and `/preview` endpoints are added to the observability middleware's `_QUIET_ROUTES` allowlist ŌĆö they're polled by the on-screen view + iframe respectively, and would otherwise spam the trace log per PR #58.

## Versioning + deprecation

No URL-shape break, no removal of `/report`. The HTML/MD content rendered by `/export?format=html` and `/export?format=md` IS visibly different (new layout) but the URL contract is preserved. Operators with bookmarks / saved exports get a new shape on next render ŌĆö Akhilesh confirmed this is the desired behaviour (the old shape was not client-shareable).
